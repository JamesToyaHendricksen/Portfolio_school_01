1 システム概要

システムの目的 必須
このシステムが解決する課題と提供する価値

LifeShield AI の管理システムは、個人PCユーザーが利用し、危険イベントの確認、監視設定、レポート閲覧を行える仕組みです。AIによる検知結果を分かりやすく整理し、安心してデジタル生活を送れる価値を提供します。

システム構成図 必須
フロントエンド ↔ バックエンド ↔ データベースの関係を図示
[ユーザー]
   ↓
[ブラウザ]
   ↓ HTTPリクエスト
[フロントエンド]
HTML / CSS / JavaScript
(Thymeleaf)
   ↓
[Spring Boot]
   ↓
[Controller]
   ↓
[Service]
   ↓
[Repository]
   ↓
[データベース]
MySQL / H2

データの流れ
1. ユーザーがブラウザで操作する
2. ブラウザから Spring Boot へリクエストを送る
3. Controller がリクエストを受け取る
4. Service が業務処理を行う
5. Repository がデータベースへアクセスする
6. データベースの結果を Repository → Service → Controller の順で返す
7. Thymeleaf で画面を生成し、ブラウザへレスポンスを返す
8. ブラウザに結果が表示される

4 機能詳細仕様

F-001 ログイン

機能概要 必須

メールアドレス、パスワード、ログインボタンを中央に配置する。入力後に認証を行い、成功時はダッシュボードへ遷移する。未入力、形式不正、認証失敗時は同画面にエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
メールアドレスString○必須、メール形式、255文字以内認証ID
パスワードString○必須、8文字以上、100文字以内マスク表示
ログインボタンAction○入力値が有効な場合のみ実行成功時はダッシュボードへ

F-002 ログアウト

機能概要 必須

ヘッダー右上のログアウト導線からセッションを破棄し、ログイン画面へ戻す。確認ダイアログを設ける場合は、OKで実行、キャンセルで元画面に留まる。失敗時は同画面でエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
ログアウトボタンAction○押下時にログアウト処理ヘッダー配置
確認OKAction△確認ありの場合のみセッション破棄
確認キャンセルAction△処理中止元画面へ戻る

F-003 危険イベント一覧表示

機能概要 必須

上部に検索条件、中央に一覧テーブル、下部にページネーションを配置する。危険イベントを取得して発生日、種別、危険度、状態、操作を表示し、詳細押下で詳細画面へ遷移する。取得失敗時は一覧上部にエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
キーワードString—100文字以内検索条件
危険度・状態String—定義済み値のみ絞り込み
詳細ボタンAction○対象IDが存在すること詳細画面へ

F-004 危険イベント詳細表示

機能概要 必須

概要情報、検知内容、危険理由、推奨対応を確認できる詳細画面を表示する。管理者には編集、削除ボタンを表示し、一般ユーザーには閲覧のみを許可する。対象が存在しない場合はエラーを表示して一覧へ戻す。

入力項目一覧 必須
項目名データ型必須バリデーション備考
イベントIDInteger○1以上、対象データ存在URLパラメータ
編集ボタンAction△管理者のみ編集画面へ
削除ボタンAction△管理者のみ削除確認へ

F-005 危険イベント新規登録

機能概要 必須

イベント種別、発生日、危険度、状態、対象チャネル、検知内容などを入力できるフォームを配置する。保存時に入力チェックを行い、成功時は一覧画面へ戻る。形式不正や必須未入力時は同画面にエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
イベント種別String○必須、50文字以内プルダウン可
発生日Date○必須、未来日不可日付入力
検知内容String○必須、500文字以内テキストエリア
添付ファイルFile—許可形式、上限サイズ以内任意

F-006 危険イベント編集

機能概要 必須

既存データを初期表示した編集フォームを配置し、管理者が内容を更新できるようにする。更新押下後に入力チェックを行い、成功時は詳細画面へ戻る。対象データが存在しない場合や更新失敗時はエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
イベントIDInteger○対象データが存在すること更新対象
危険度String○高・中・低のみ選択式
状態String○未対応・確認中・対応済みのみ選択式
更新ボタンAction○入力値が有効な場合のみ更新詳細画面へ戻る

F-007 危険イベント削除

機能概要 必須

詳細画面から削除操作を行い、確認ダイアログで確定後に対象イベントを削除する。成功時は一覧画面へ戻り、失敗時は詳細画面上部にエラーを表示する。管理者のみが実行できる。

入力項目一覧 必須
項目名データ型必須バリデーション備考
イベントIDInteger○対象データが存在すること削除対象
削除ボタンAction○管理者のみ確認ダイアログ表示
確認OKAction○押下時に削除処理一覧へ戻る

F-008 監視設定更新

機能概要 必須

メール監視、SNS監視、通信監視、通知設定を変更できるフォームを配置する。現在設定を初期表示し、保存時に入力チェックを行って設定を更新する。正常時は完了メッセージを表示してダッシュボードへ戻る。

入力項目一覧 必須
項目名データ型必須バリデーション備考
メール監視・SNS監視・通信監視Boolean○ON/OFF のいずれかトグル操作
通知メールアドレスString—メール形式、255文字以内任意
保存ボタンAction○入力値が有効な場合のみ保存ダッシュボードへ戻る

F-009 レポート閲覧

機能概要 必須

レポート一覧画面で期間や種別による絞り込みを行い、一覧から詳細へ遷移できる。画面上部に検索条件、中央にテーブル、各行に詳細ボタンを配置する。データ取得失敗時はエラーメッセージを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
キーワードString—100文字以内検索条件
期間開始日・終了日Date—日付形式、終了日は開始日以降絞り込み
詳細ボタンAction○対象レポートが存在すること詳細画面へ遷移

F-010 入力バリデーション

機能概要 必須

ログイン、登録、編集、検索など全入力画面で共通に動作し、必須、形式、文字数、期間整合性を検証する。エラーがある場合は項目下にメッセージを表示し、正常時のみ本処理へ進む。想定外エラー時は汎用メッセージを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
メールアドレスString画面による必須、メール形式、255文字以内共通項目
日付項目Date画面による日付形式、期間整合性検索・登録で使用
ファイル項目File—許可形式、上限サイズ添付用

F-011 危険イベント検索

機能概要 必須

危険イベント一覧画面上部の検索条件エリアからキーワード、危険度、状態、期間で絞り込む。検索結果は同画面に再表示し、条件クリアで初期表示に戻す。日付不正や期間逆転時はエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
キーワードString—100文字以内部分一致検索
開始日・終了日Date—日付形式、終了日は開始日以降期間検索
検索・条件クリアAction○押下時に再表示一覧画面内で動作

F-012 ソート

機能概要 必須

危険イベント一覧画面で発生日、危険度、状態などを昇順・降順に並び替える。列見出しやソートボタンから操作し、現在の並び順が分かる表示を行う。不正なソート値が渡された場合は初期順に戻す。

入力項目一覧 必須
項目名データ型必須バリデーション備考
ソート項目String—定義済み項目のみ発生日、危険度など
ソート順String—asc / desc のみ昇順・降順
並び替え操作Action○押下時に再取得一覧再表示

F-013 ページネーション

機能概要 必須

危険イベント一覧画面下部にページ番号、前へ、次へを配置し、指定ページ分だけデータを表示する。検索条件やソート条件を維持したままページを切り替える。不正なページ番号指定時は先頭ページへ戻す。

入力項目一覧 必須
項目名データ型必須バリデーション備考
ページ番号Integer—1以上の整数対象ページ
表示件数Integer—定義済み件数のみ固定件数想定
前へ・次へAction○前後ページが存在すること一覧画面内で移動

F-014 レポート検索

機能概要 必須

レポート一覧画面でキーワード、期間、レポート種別による検索を行う。検索結果は同画面に再表示し、条件クリアで初期表示に戻す。日付不正や期間逆転時はエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
キーワードString—100文字以内部分一致
開始日・終了日Date—日付形式、終了日は開始日以降期間検索
レポート種別String—定義済み種別のみプルダウン可

F-015 CSV出力

機能概要 必須

危険イベント一覧画面、レポート一覧画面の上部に CSV出力ボタンを配置し、現在の検索条件に合うデータを CSV ファイルとしてダウンロードさせる。出力失敗時は一覧画面上でエラーメッセージを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
出力対象条件String—現在の検索条件を使用ユーザー入力なし
CSV出力ボタンAction○押下時にCSV生成同画面のままダウンロード

F-016 ファイルアップロード

機能概要 必須

危険イベント登録画面、編集画面で添付ファイルをアップロードできる。選択したファイルに対し形式とサイズを検証し、登録または更新処理とあわせて保存する。許可外形式やサイズ超過時は同画面でエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
添付ファイルFile—許可形式、上限サイズ以内任意入力
再選択ボタンAction—別ファイルを選択可能差し替え用
削除チェックBoolean—既存添付削除時のみ使用編集画面で利用

F-017 メール通知

機能概要 必須

監視設定画面で通知ON/OFF、通知先メールアドレス、通知対象を設定する。保存後は設定内容を更新し、危険イベント検知時に対象ユーザーへメール送信する。形式不正や保存失敗時は設定画面にエラーを表示する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
通知有無Boolean○ON/OFF のいずれかトグル操作
通知先メールアドレスString—メール形式、255文字以内任意入力
通知対象String—定義済み通知種別のみイベント種別指定

5 非機能要件

セキュリティ要件 必須
認証・認可・データ保護に関する要件

・認証方式：Spring Security を利用し、ログイン認証と認可を行う。未認証ユーザーはログイン画面へリダイレクトする。

・パスワードの暗号化方式：パスワードは平文で保持せず、BCrypt でハッシュ化して保存する。

・CSRF対策：Spring Security の CSRF 保護を有効にし、フォーム送信時にトークンを検証する。

・XSS対策：入力値を画面表示する際はエスケープ処理を行い、Thymeleaf の標準エスケープを利用する。

・SQLインジェクション対策：Repository 経由でデータアクセスし、プレースホルダや ORM を利用して動的SQLの直接連結を避ける。

・セッション管理：ログイン成功時にセッションを発行し、ログアウト時に破棄する。セッションタイムアウトを設定し、不正利用を防ぐ。

対応環境 必須
対応ブラウザ・デバイス

・対応ブラウザ：Chrome（最新版）/ Firefox（最新版）/ Edge（最新版） ・対応デバイス：PC / タブレット / スマートフォン （自分の実際の対応範囲に合わせて編集してください）
